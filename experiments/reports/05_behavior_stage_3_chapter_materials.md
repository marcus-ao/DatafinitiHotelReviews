# 行为实验第三阶段论文材料汇总

更新时间：2026-03-31

本文件用于把当前已经冻结的 `E3 / E4 / E5` 正式结果整理成可直接复用到论文正文中的材料。当前最稳妥的主结论是：`Qwen3.5-2B` 只适合作为行为层弱基线，`Qwen3.5-4B` 已经足以作为正式行为主模型，而 `E5` 明确证明了“中文交互 + 结构化英文检索表达桥接”的必要性。

## 1. 一句话摘要

在固定检索后端 `aspect_main_no_rerank` 和 `fallback=false` 的前提下，`Qwen3.5-4B` 已经可以稳定完成偏好解析与澄清触发，且结构化英文检索表达明显优于中文直检，因此当前行为层主线已经具备进入论文正文的条件。

- 即使桥接后，`avoid` 与 `quiet_sleep` 仍偏弱，这说明剩余瓶颈已更多转向证据覆盖，而不只是语言桥接。

## 5. 代表性案例

### 案例 A：`E3` 的 `value` 负向约束仍是残留边界

- Query：`q048`
- 原句：`我在Seattle想住得安静一点，但不要性价比太差的酒店。`
- gold：`avoid_aspects=["value"]`
- `4B` 预测：`avoid_aspects=[]`, `unsupported_requests=["value"]`
- 含义：`4B` 已经不再系统性漏槽位，但在少数 `value` 负向约束上仍会把“避免项”错看成 unsupported。

### 案例 B：`E3` 的 focus + avoid 同方面冲突仍最脆弱

- Query：`q062`
- 原句：`我想在New Orleans找一家性价比很好，但又最好别太强调性价比的酒店。`
- gold：`focus=["value"], avoid=["value"]`
- `4B` 预测：`focus=["value"], avoid=[]`, `unsupported=["budget"]`
- 含义：同一方面的双向约束是当前 `E3` 最难的边界类型，也是后续 `E10 / PEFT` 最值得重点观察的行为点。

### 案例 C：`E4` 的边界误澄清已经收缩到极少数样本

- 边界例：`q013`
- 原句：`我在Chicago想住得安静一点，但不要安静睡眠太差的酒店。`
- gold：不需要澄清
- 规则组与 `4B` 都把它误判为 conflict
- 含义：这类“同一方面的正负修饰”在当前 gold 口径下仍容易被看成冲突，是 `E4` 目前最突出的残留边界。

### 案例 D：`4B` 已经开始在边界句上优于规则组

- 对比例：`q043`
- 原句：`我在San Francisco想住得安静一点，但不要安静睡眠太差的酒店。`
- gold：不需要澄清
- 规则组：误判为 conflict
- `4B`：正确判为 `none`
- 含义：`4B` 已经不只是“能跑通”，而是开始在部分边界句上表现出比规则组更好的语义判断。

### 案例 E：`E5` 证明桥接不是可有可无的润色项

- Query：`q043`
- target_aspect：`quiet_sleep`
- 中文直检：`nDCG@5 = 0.0000`
- 结构化英文检索：`nDCG@5 = 0.8282`
- 含义：即使 query 本身不复杂，若直接用中文原句去英文检索后端，仍可能完全取不到正确方面证据；桥接带来的不是轻微优化，而是能否稳定命中证据的差异。

## 6. 统一章节结论

当前行为层最稳妥的论文表述应固定为：

1. `Qwen3.5-2B` 作为行为层弱基线保留，用于说明轻量模型能力下界。
2. `Qwen3.5-4B` 已经足够稳定地支撑 `E3 / E4` 正式论文结果，是当前默认正式行为模型。
3. 中文交互外观必须通过“结构化归一 -> 英文检索表达”桥接到英文检索后端，否则证据命中会明显下降。
4. 当前行为层结果已经足以支撑论文主线；是否补 `9B` 只影响附录强度，不改变现有主结论。
