# 第四章结果解释与正文转写材料（E1-E4）

更新时间：2026-04-07

本文件用于为论文第四章“酒店评论知识库构建与推荐工作流基础”提供更接近正文风格的写作底稿。第四章的核心任务不是证明某个单独模块在所有指标上都达到了最优，而是通过 `E1-E4` 四组实验说明：当前知识库与推荐工作流基础设施已经达到“足以支撑后续正式实验”的可用水平。因此，本章的写法应当突出“前置条件验证”这一角色，强调可用性、稳定性与边界诚实性，而不应误写成全文的核心贡献章。

## 1. 第四章的章节角色与总体写法

从全文结构看，第四章承担的是承前启后的基础支撑作用。它要回答的问题并不是“知识库构建方法本身是否已经做到最强”，而是“现有数据底座、候选生成链路、偏好解析模块与澄清模块，是否已经足够稳定，以至于第五章和第六章中关于 RAG 与 PEFT 的正式比较不会建立在明显失效的前提之上”。因此，本章的所有实验都应围绕“是否可用、是否稳定、是否足以支撑后续实验”来组织叙述。换言之，第四章更接近于一组严格的前置条件检验，而不是独立的核心创新实验。

## 2. E1：方面分类与情感标注可靠性

`E1` 的作用在于验证知识库中 aspect 标签与 sentiment 标签是否具有足够可靠性，使得后续 retrieval 与 grounded recommendation 不会从一开始就建立在错误的评论结构化信息之上。对于这一实验，正文不宜写成“某种方案全面优于其他方案”的竞赛式叙述，而应将重点放在“哪种方案更适合作为下游系统的工程底座”上。

当前结果最需要诚实保留的一点是：`hybrid` 并不在所有指标上全面优于 `rule-only`。更稳妥的表述应当是，`hybrid` 的优势主要体现在困难样本上的覆盖与稳健性，而不在于整体 `macro-F1` 的绝对领先。这意味着，本研究选择 `hybrid` 作为知识库标注方案，并不是因为它在所有统计意义上都“最好”，而是因为它在复杂样本和工程落地场景下更能提供稳定的结构化标签支持。与此同时，`zeroshot` 的较低表现不应被简单粗暴地描述为“效果差”，而应解释为：在当前任务设定与语料条件下，纯零样本路线尚不能提供足够稳定的方面级标注能力。至于 sentiment 侧精度的不足，也应当被简洁承认，但不必在本章扩写为一个独立研究问题，只需说明它是后续 retrieval / generation 结果中需要保持意识的弱噪声来源即可。

因此，在第四章正文中，`E1` 最稳妥的结论可以写成：当前方面与情感标注方案已经达到知识库工程可用水平；虽然 `hybrid` 并非在所有全局指标上都优于 `rule-only`，但它在困难样本覆盖与整体工程稳健性上更适合作为下游 retrieval 与 recommendation 工作流的基础支撑。

## 3. E2：候选酒店缩圈可行性验证

`E2` 的作用，是验证候选酒店缩圈阶段是否能够在 test hotel 范围内提供一个规模可控且质量足够的候选集合。这里必须避免一个常见写作错误：把候选层 proxy 指标直接当成最终推荐质量的证据。`E2` 的实验层级应被严格限定在“候选缩圈是否可用”这一问题上，而不是“最终推荐是否已经最优”。

基于当前结果，更合适的写法应是：候选酒店筛选策略已经能够在 test hotel 范围内形成一个规模适中、命中能力可接受的候选集合，因此后续 retrieval 与 generation 阶段无需从全量酒店直接搜索。也就是说，`E2` 的价值在于为第五章和第七章提供一个稳定的上游候选空间，而不是直接证明最终推荐本身的优劣。

## 4. E3：偏好解析可靠性

`E3` 的任务是证明偏好解析模块足够稳定，不会成为后续正式实验中的主要系统性误差来源。正文中最重要的不是夸大该模块“已经达到完美”，而是说明它的稳定性足以让后续差异主要归因于 RAG / PEFT 路线本身，而不是归因于上游偏好抽取错误。

根据当前 canonical 结果，`E3` 的 `Exact-Match` 水平已经处于较高区间，这意味着主模型在结构化偏好解析上具备稳定可用性。更准确的正文表述是：当前偏好解析模块已经足够可靠，因此后续 retrieval 与 generation 实验中的主要差异，应解释为增强路线本身带来的变化，而非偏好解析不稳定所造成的假象。与此同时，也应保留一条必要的审慎说明：这里的结论是“足够稳定”，而不是“绝对无误”；本研究并不需要也不应在第四章中把 E3 写成大模型行为能力的全面胜利故事。

## 5. E4：澄清触发可靠性

`E4` 是第四章中最容易被写歪的一组结果。当前 canonical summary 显示，系统在澄清判断上的总体准确率较高，且过度澄清率较低，但 `precision / recall / f1` 在当前切片中均为 `0`。如果不加解释，极易被误写成“系统完全不会澄清”。然而，从当前结果结构来看，更合理的解释是：在现有切片下，有效正例极少，导致 PR/F1 指标缺乏信息量，而不是系统完全失去澄清能力。

因此，第四章对 `E4` 的写法必须明确转向：以 `clarification_accuracy`、`over_clarification_rate` 和 `under_clarification_rate` 作为主结论支撑，而将 PR/F1 视为当前切片下的低信息量指标。最稳妥的论述方式是：当前澄清模块在工程上已经足够可用，它总体上能够避免过度打断推荐流程，同时也没有表现出明显的漏澄清风险。PR/F1 的零值应被视为切片结构导致的评测局限，而不是系统功能失效的证据。

## 6. 第四章统一收束段

综合 `E1-E4` 的结果，可以认为当前系统的知识库标注、候选酒店缩圈、偏好解析与澄清触发四个基础环节均已达到“足以支撑后续正式实验”的可用水平。尽管这些模块并非在所有细粒度指标上都达到最优或完全无缺陷，但它们已经不再构成后续 RAG 与 PEFT 比较中的主要系统性风险。因此，第五章与第六章中的结果差异更应被解释为增强路线本身的影响，而不是基础设施不稳定所导致的偏差。

## 7. 第四章写作时需要避免的表述

为了保持本章的学术严谨性，后续正式转写时应避免以下几类写法：

1. 把第四章写成“本研究的核心贡献章”；  
2. 把 `E1` 写成 `hybrid` 全面优于 `rule-only`；  
3. 把 `E2` 的候选命中写成最终推荐质量证据；  
4. 把 `E3` 写成大模型行为能力全面优于所有其他路线；  
5. 把 `E4` 的 `PR/F1 = 0` 写成“系统不会澄清”。
